# CalendarTaskMate - UI/UX 가이드

**최종 업데이트:** 2026년 7월 2일  
**디자인 철학:** 단순함, 명확함, 시각적 위계  
**기본 색상:** Tailwind CSS Default Palette + 프로젝트 카스텀

---

## 1. 디자인 철학 & 원칙

### 1.1 핵심 원칙
1. **시간이 주축** — 모든 UI는 시간 개념을 중심으로 설계
2. **시각적 명확함** — 색상, 아이콘, 텍스트로 상태를 한눈에 파악
3. **최소 클릭** — 자주 쓰는 기능은 2클릭 이내
4. **반응형 첫 번째** — 데스크톱(1280px+)부터 모바일(375px+) 모두 지원
5. **접근성** — WCAG 2.1 Level AA, 키보드 네비게이션 필수

### 1.2 시각적 위계
```
1순위: 캘린더 그리드 (전체 영역의 60%)
2순위: 필터/검색 (우측 20%)
3순위: 백로그 (하단 20%)
4순위: 헤더/네비게이션 (상단 고정)
```

## 4. 컴포넌트 라이브러리

### 4.1 shadcn/ui 컴포넌트 사용 목록

**설치할 컴포넌트:**
```bash
npx shadcn@latest add button input card dropdown-menu select dialog
npx shadcn@latest add tabs badge alert checkbox radio-group
npx shadcn@latest add popover calendar date-picker time-picker
npx shadcn@latest add progress skeleton alert-dialog sheet
```

**사용 패턴:**
```tsx
// Button
<Button variant="default" size="md">저장</Button>
<Button variant="secondary">취소</Button>
<Button variant="ghost">옵션</Button>
<Button variant="destructive">삭제</Button>

// Card
<Card className="p-4">
  <CardHeader>제목</CardHeader>
  <CardContent>내용</CardContent>
</Card>

// Badge
<Badge variant="default">배정됨</Badge>
<Badge variant="secondary">낮음</Badge>
```

### 4.2 커스텀 컴포넌트

#### 4.2.1 TimeBlock
시간 단위 슬롯 표시

```tsx
<TimeBlock
  time="14:00"
  duration="1h"
  task={{
    id: 123,
    title: "마케팅 이미지",
    project: "Project A",
    color: "#3B82F6"
  }}
  onClick={() => navigate(`/tasks/123`)}
  onRemove={() => handleRemove()}
/>
```

**스타일:**
- 배경: 프로젝트 색 (투명도 80%)
- 테두리: 프로젝트 색 (2px)
- 패딩: 8px (내부)
- 모서리: 4px (둥근 모서리)

#### 4.2.2 TaskCard
백로그에 표시되는 작업 카드

```tsx
<TaskCard
  task={task}
  isDragging={isDragging}
  onSelect={() => navigate(`/tasks/${task.id}`)}
/>
```

**레이아웃:**
```
┌─────────────────────┐
│ 📋 마케팅 이미지     │
│ Client: Project A  │
│ 예상: 1h           │
└─────────────────────┘
```

- 높이: 80px
- 너비: 100% (부모)
- 마우스 hover: 그림자 증가, scale 1.02

#### 4.2.3 CalendarGrid
주간 캘린더 그리드 (핵심)

```tsx
<CalendarGrid
  week={week}
  events={events}
  assignments={assignments}
  onDrop={handleDrop}
  onCellClick={handleCellClick}
/>
```

**구조:**
```
────────────────────────────────────────────
   월     화     수     목     금     토     일
────────────────────────────────────────────
06:00  [ ]   [ ]   [ ]   [ ]   [ ]   [ ]   [ ]
07:00  [ ]   [ ]   [ ]   [ ]   [ ]   [ ]   [ ]
...
22:00  [ ]   [ ]   [ ]   [ ]   [ ]   [ ]   [ ]
```

- 셀 크기: 80px (높이) × (width/7) (너비)
- 셀 배경: 화이트 (또는 매우 밝은 회색)
- 셀 테두리: 1px 회색
- 행 높이: 일정 (60px)

#### 4.2.4 EventBadge
이벤트 표시 (회의 등)

```tsx
<EventBadge
  event={{
    title: "팀 미팅",
    time: "10:00~11:00"
  }}
  onEdit={() => {}}
  onDelete={() => {}}
/>
```

**스타일:**
- 배경: Gray 300
- 텍스트: Gray 900
- 아이콘: 📅
- 테두리: None (배경으로 충분)

#### 4.2.5 EmptyState
콘텐츠 없음 상태

```tsx
<EmptyState
  icon="📋"
  title="작업이 없습니다"
  description="아래 버튼으로 첫 작업을 만들어보세요."
  action={<Button>새 작업 추가</Button>}
/>
```

**레이아웃:**
```
        📋
     작업이 없습니다
  아래 버튼으로 첫 작업을...
    
    [새 작업 추가]
```

- 높이: 300px (최소)
- 아이콘: 64px
- 가운데 정렬

#### 4.2.6 LoadingSpinner
로딩 상태

```tsx
<LoadingSpinner size="md" message="캘린더를 불러오는 중..." />
```

**옵션:**
- size: sm (20px), md (40px), lg (60px)
- message: 선택사항

#### 4.2.7 FilterPanel
필터 패널

```tsx
<FilterPanel
  filters={filters}
  onFilterChange={handleFilterChange}
  onReset={handleReset}
/>
```

**필터:**
- 상태 (라디오 또는 탭)
- 클라이언트 (체크박스)
- 날짜 범위 (달력)
- 우선순위 (체크박스)

## 5. 화면별 와이어프레임 & 상세

### 5.1 헤더 (모든 페이지)

```
┌────────────────────────────────────────────────────────┐
│ CalendarTaskMate  🔍 검색...        프로필▼  🌙  ≡      │
└────────────────────────────────────────────────────────┘
```

**요소:**
- 좌측: 로고 (텍스트 + 아이콘)
- 중앙: 검색창 (투명 배경, 포커스 시 보더)
- 우측: 프로필 드롭다운, 다크모드 토글, 메뉴 (모바일)

**높이:** 56px  
**고정:** position: sticky (top: 0)

### 5.2 사이드바 (데스크톱)

```
┌─────────────┐
│ 📊 대시보드   │

│ 📋 작업 목록  │
```

**너비:** 240px  
**항목 높이:** 48px  
**선택됨:** 배경 Blue 100, 텍스트 Blue 700  
**호버:** 배경 Gray 100

**모바일:** 헤더 메뉴 (≡) → 드롭다운 또는 바닥시트

## 7. 반응형 전략

### 7.1 브레이크포인트 (Tailwind)

| 이름 | 너비 | 레이아웃 |
|------|------|----------|
| sm | 640px | 1열 (모바일) |
| md | 768px | 1-2열 (태블릿) |
| lg | 1024px | 2-3열 (랩탑) |
| xl | 1280px | 풀 3열 (데스크톱) |

### 7.2 모바일 (375px ~ 640px)

**대시보드:**
```
헤더
─────────────
차트 1
차트 2
차트 3 (스택)
─────────────
```

**캘린더:**
- 사이드바: 숨김 (헤더 메뉴로 변경)
- 백로그: 하단 시트 (드래그 가능)
- 캘린더: 단일 열 (일간 뷰로 변경)
- 필터: 우측에서 모달로 변경

**작업 목록:**
- 테이블 → 카드 뷰로 변경
- 텍스트 크기 13px로 감소
- 한 줄 요약만 표시

### 7.3 태블릿 (768px ~ 1024px)

**캘린더:**
- 사이드바 여전히 표시 (120px로 축소)
- 필터 패널: 축소 (아이콘만)
- 캘린더: 2일씩 표시 (또는 주간 축소)

### 7.4 데스크톱 (1280px+)

**전체 레이아웃:**
- 사이드바 (240px)
- 메인 콘텐츠 (calc(100% - 240px))
- 필터 패널 또는 상단 필터바

## 8. 상호작용 및 애니메이션

### 8.1 기본 애니메이션

```css
/* Transition 기본값 */
transition: all 0.2s ease-in-out;

/* 색상 변경 */
transition: background-color 0.15s ease-in, 
            color 0.15s ease-in;

/* 크기 변경 */
transition: transform 0.2s ease-in-out;

/* 투명도 */
transition: opacity 0.15s ease-in;
```

### 8.2 상호작용 상태

#### Hover (버튼)
```css
background-color: lighten(current-color, 5%);
box-shadow: 0 2px 8px rgba(0,0,0,0.1);
transform: translateY(-2px);
```

#### Focus (폼 입력)
```css
outline: 2px solid blue-500;
outline-offset: 2px;
box-shadow: 0 0 0 3px blue-100;
```

#### Active (클릭)
```css
transform: scale(0.98);
background-color: darken(current-color, 5%);
```

#### Disabled
```css
opacity: 0.5;
cursor: not-allowed;
```

### 8.3 페이지 전환

- Fade-in (200ms)
- 스크롤 자동 상단 이동

### 8.4 드래그 애니메이션

- 드래그 시작: scale(0.95), opacity(0.7)
- 드래그 중: smooth tracking
- 드롭 후: smooth placement (100ms)

## 9. 접근성 가이드

### 9.1 색상 대비

**WCAG AA 기준:**
- 기본 텍스트: 4.5:1 대비율
- 큰 텍스트 (18px+): 3:1 대비율

**테스트:** WebAIM Contrast Checker

### 9.2 키보드 네비게이션

| 키 | 기능 |
|----|------|
| Tab | 다음 요소로 이동 |
| Shift+Tab | 이전 요소로 이동 |
| Enter | 버튼/링크 활성화 |
| Space | 체크박스/라디오 토글 |
| Escape | 모달 닫기, 드래그 취소 |
| Arrow Keys | 메뉴/탭 네비게이션 |

### 9.3 ARIA 라벨

```tsx
// 버튼
<button aria-label="새 작업 추가">+</button>

// 폼 입력
<input aria-label="작업 제목" type="text" />
<label htmlFor="title">작업 제목</label>

// 아이콘
<span aria-label="진행중">⏳</span>

// 드롭다운
<div role="listbox" aria-label="상태 필터">
  <div role="option" aria-selected="true">배정됨</div>
</div>
```

### 9.4 화면 리더

- 모든 아이콘에 alt/aria-label
- 상태 변경 시 live region 사용
- 폼 에러 메시지 연결 (aria-describedby)
