**최종 업데이트:** 2026년 7월 2일  
**상태:** 📋 PRD 확정  
**문서 버전:** 1.0

## 1. 서비스 개요

### 1.1 비전
캘린더 그리드를 주 인터페이스로 하여, 회의와 딥 워크를 한눈에 볼 수 있고, 미배정 작업을 드래그 앤 드롭으로 시간 블록에 할당할 수 있는 **개인용 시간블로킹 작업 관리 도구**.

### 1.2 핵심 가치 제안
1. **캘린더 우선 인터페이스** — 실제 일정(회의, 이벤트)과 작업을 같은 그리드에 표시
2. **드래그 기반 시간블로킹** — 작업을 시간 블록으로 끌어놓아 자동으로 할당
3. **로컬 우선 아키텍처** — SQLite 캘린더로 완전 자체 완결, 외부 연동 없음
4. **프리랜서 특화** — 클라이언트별 색 구분, 청구 시간 추적
5. **저가/무료 모델** — Reclaim.ai ($15+/월)의 대체 선택지

### 1.3 대조 경쟁 분석

| 기능 | Todoist | Reclaim.ai | Google Tasks | CalendarTaskMate |
|-----|---------|-----------|-------------|-----------------|
| 캘린더 이벤트 표시 | ✗ | ✓ | ✗ | ✓ |
| 드래그 시간블로킹 | ✗ | ✓ | ✗ | ✓ |
| 로컬 우선 | ✗ | ✗ | ✗ | ✓ |
| 프리랜서 기능 | ✗ | ✓ | ✗ | ✓ |
| 가격 | $3~5/월 | $15+/월 | 무료 | **무료 오픈소스** |

## 2. 타겟 페르소나

### 2.1 주요 타겟
**"바쁜 프리랜서 & PM"**
- 연령: 25~45세
- 특징: 일정이 빈번 (주 5~10건), 여러 프로젝트 병행, 딥 워크/포커스 시간 보호 필수
- 통증점:
  - Todoist는 캘린더 뷰가 약해서 실제 일정과 작업을 동시에 보기 어려움
  - Google Calendar에 이벤트만 있고, 할 일 앱과 분리됨
  - Reclaim.ai는 가격이 비싸고 AI 기능이 오버스펙
- 원하는 것: 단순하게 캘린더 그리드에 작업을 끌어놓고, 자동으로 시간 할당

### 2.2 보조 타겟
- **소규모 팀 PM**: 팀원 개인이 사용
- **학생/연구자**: 강의 일정과 과제를 시간 블로킹
- **온라인 강사**: 수업 시간과 콘텐츠 제작 시간 구분

## 3. 킬러 기능 (핵심 차별화)

### 3.1 기능 1: 캘린더 그리드 기반 시간블로킹
- **설명**: 주간/월간 캘린더 그리드에 미배정 작업을 드래그하면, 시작·종료 시간 팝업이 나타나고 자동 할당
- **체감**: Todoist처럼 "오늘 할 일" 리스트에서 빼내, 실제 시간표에 적합하게 정렬
- **차별화**: 기존 캘린더 앱(Google, Outlook)은 수동 입력만 가능. Reclaim.ai는 AI가 자동 배치하지만 가격 비쌈

### 3.2 기능 2: 실제 일정 + 작업 통합 뷰
- **설명**: 로컬 캘린더에 저장된 회의·이벤트와 작업을 같은 셀에 색으로 구분해 표시
- **체감**: "오전 10시에 회의, 11시~1시에 딥 워크" 한눈에 파악
- **차별화**: Todoist 캘린더 뷰는 작업만 표시, 이벤트는 별도로 봐야 함

### 3.3 기능 3: 프리랜서 청구 시간 추적
- **설명**: 클라이언트별로 색을 정하고, 그 시간만 합산해서 청구 가능한 시간 계산
- **체감**: "A 프로젝트에 이번 주 12시간 할당됨 → 청구 가능" 자동 계산
- **차별화**: Reclaim.ai도 있지만, 로컬 우선이라 더 빠르고 비용 절감

## 4. 기능 요구사항 (MoSCoW)

### 4.1 Must-Have (MVP 필수)

#### MH-1: 캘린더 그리드 표시
- **설명**: 주간 캘린더를 시간 단위 그리드로 표시 (월~일, 시간대별)
- **세부**:
  - [ ] 시간 단위 슬롯 (30분 또는 1시간 간격)
  - [ ] 월~일 열, 시간 행 구성
  - [ ] 스크롤 가능 (전체 24시간 또는 6~22시 근무시간만)
- **테스트**: "2026-07-06 월요일부터 7월 12일 일요일까지 1주일 캘린더가 보인다"

#### MH-2: 작업 CRUD
- **설명**: 작업 생성, 읽기, 수정, 삭제
- **세부**:
  - [ ] 작업 생성: 제목, 설명, 예상 시간 (기본 1시간), 클라이언트/프로젝트 선택
  - [ ] 작업 목록 페이지: 미배정 작업 목록 (백로그)
  - [ ] 작업 상세 페이지: 제목, 설명, 시간, 상태 (미배정/배정/진행중/완료) 수정
  - [ ] 작업 삭제 (휴지통 또는 즉시 삭제)
- **테스트**: "작업 생성 후 백로그에 나타나고, 상세 페이지에서 수정 가능하다"

#### MH-3: 드래그 앤 드롭 시간블로킹
- **설명**: 백로그의 작업을 캘린더 그리드 슬롯에 드래그하면 시간 할당
- **세부**:
  - [ ] 드래그 시작: 백로그/미배정 영역의 작업 선택
  - [ ] 드래그 중: 타겟 슬롯 강조 (hover effect)
  - [ ] 드롭 후: 시간 할당 팝업 (시작 시간, 종료 시간 자동 설정)
  - [ ] 할당된 작업: 캘린더 셀에 표시 (프로젝트 색)
  - [ ] 드래그 취소: ESC or 드롭 영역 외 drop
- **라이브러리**: dnd-kit 또는 react-beautiful-dnd (v16 호환성 확인 필수)
- **테스트**: "백로그의 '마케팅 이미지 제작' 작업을 월요일 14:00~16:00 슬롯에 드래그하면 할당된다"

#### MH-4: 캘린더 이벤트 표시 (로컬)
- **설명**: SQLite 캘린더에 저장된 회의, 이벤트를 같은 그리드에 표시
- **세부**:
  - [ ] 로컬 이벤트 저장 스키마 (제목, 시작·종료, 설명)
  - [ ] 이벤트 생성/편집 (캘린더 셀 더블클릭 또는 "+ 이벤트" 버튼)
  - [ ] 이벤트 표시: 작업과 다른 색 (예: 회의=회색, 작업=컬러)
  - [ ] iCal 임포트 (v1.1에서 구현, MVP는 수동 입력만)
- **테스트**: "이벤트 '팀 미팅' 생성 후 월요일 10:00~11:00에 표시된다"

#### MH-5: 완료율/진행상태 표시
- **설명**: 할당된 작업의 완료 상태를 진행바 또는 체크박스로 표시
- **세부**:
  - [ ] 작업 상태: 미배정 → 배정됨 → 진행중 → 완료
  - [ ] 캘린더 셀의 작업 카드에 상태 아이콘 (미완=점선, 진행중=시계, 완료=체크)
  - [ ] 완료율 진행바 (선택사항: 캘린더 셀 상단에 %로 표시)
  - [ ] 클릭해서 상태 변경 (작업 카드 클릭 → 상태 버튼)
- **테스트**: "할당된 작업을 클릭해서 '진행중' → '완료'로 변경하면 체크 아이콘이 나타난다"

#### MH-6: 검색 & 필터
- **설명**: 작업과 이벤트를 빠르게 찾기
- **세부**:
  - [ ] 검색: 제목, 설명 텍스트 검색 (캘린더 우측 검색창)
  - [ ] 필터: 클라이언트/프로젝트 선택 (체크박스 또는 드롭다운)
  - [ ] 필터: 상태 (미배정/배정/진행중/완료)
  - [ ] 필터: 날짜 범위 (주/월 선택)
  - [ ] 필터 리셋 버튼
- **테스트**: "검색창에 '마케팅'이라 입력하면 마케팅 관련 작업만 표시"

#### MH-7: 여유시간 시각화
- **설명**: 캘린더 그리드에 남은 여유시간을 색으로 구분해 표시
- **세부**:
  - [ ] 배정된 시간 vs 전체 시간 계산
  - [ ] 빈 슬롯: 밝은 회색 (배정 가능)
  - [ ] 배정된 슬롯: 프로젝트 색
  - [ ] 이벤트 슬롯: 회색 (점선 테두리)
  - [ ] 일일 요약: 하단에 "07월 07일: 배정 6시간 / 여유 2시간" 표시
- **테스트**: "월요일 여유시간이 시각적으로 구분되고, 일일 요약에 수치가 나타난다"

#### MH-8: 통계 & 대시보드
- **설명**: 주간/월간 작업량, 클라이언트별 시간 비율 시각화
- **세부**:
  - [ ] 주간 대시보드: 일별 배정 시간, 완료율 (막대 그래프)
  - [ ] 프로젝트별 시간 (원형 차트 또는 누적 막대)
  - [ ] 완료율 (전체 작업 중 완료된 작업 %)
  - [ ] 예상 마감: 현재 속도로 남은 작업 언제 완료될지 표시
- **라이브러리**: Recharts (shadcn 호환, 가벼움)
- **테스트**: "대시보드에서 이번 주 A 프로젝트가 전체의 40%, B 프로젝트가 30% 차지하는 것을 본다"

#### MH-9: 빈 상태 & 로딩/에러 처리
- **설명**: UX 안정성: 데이터 없을 때, 로딩 중, 오류 발생 시 화면
- **세부**:
  - [ ] 빈 상태 (Empty State): "작업을 추가해주세요" + 예시 이미지/텍스트
  - [ ] 로딩 상태: 스켈레톤 로더 또는 스피너
  - [ ] 에러 화면: "데이터를 불러올 수 없습니다" + 재시도 버튼
  - [ ] 백로그 비어있음: "모든 작업이 캘린더에 배정되었습니다" + 격려 메시지
- **테스트**: "처음 방문 시 빈 상태 화면이 표시되고, 작업 추가 버튼이 명확하다"

## 5. 화면 명세 (6개 이상)

### 5.1 대시보드 (Dashboard)
**경로**: `/`  
**목적**: 주간 요약, 오늘의 일정 표시  
**주요 요소**:
- 오늘의 일정 카드 ("오늘 배정된 시간: 6h / 여유: 2h")
- 주간 막대 그래프 (일별 배정 시간)
- 프로젝트별 원형 차트 (시간 비율)
- 상단 네비게이션 (주간 캘린더로 바로가기)

### 5.2 주간 캘린더 뷰 (Weekly Calendar)
**경로**: `/calendar?view=week`  
**목적**: 핵심 화면 — 시간블로킹의 중심  
**주요 요소**:
- 좌측: 시간 축 (06:00~22:00 or 고정 영역)
- 중앙: 월~일 열, 시간 행 그리드
- 우측: 필터 패널 (클라이언트 체크박스, 상태 필터)
- 미배정 작업 백로그 (하단 또는 모달로 표시)
- 드래그 가능한 작업 카드 (프로젝트 색, 제목, 예상 시간)
- 상단: "이전 주", "오늘", "다음 주" 네비게이션

### 5.3 일간 뷰 (Daily View) - Optional 초기 구현
**경로**: `/calendar?view=day&date=2026-07-07`  
**목적**: 상세한 시간 단위 계획  
**주요 요소**:
- 해당 날의 세로 타임라인 (30분 간격)
- 모든 이벤트/작업을 상세히 표시
- 빠른 메모 추가 ("이 시간에 할 일 추가")

### 5.4 작업 목록/백로그 (Task List / Backlog)
**경로**: `/tasks?status=unassigned`  
**목적**: 모든 작업 일괄 관리  
**주요 요소**:
- 필터: 상태 (미배정/배정/진행중/완료), 클라이언트, 날짜 범위
- 정렬: 제목, 시간, 생성일, 마감일
- 테이블 또는 카드 뷰:
  - 체크박스 (선택)
  - 제목 (클릭해서 상세 페이지로)
  - 클라이언트/프로젝트 (색 아이콘)
  - 예상 시간 (1h, 2h 등)
  - 상태 (배정/진행/완료)
- "+ 새 작업" 버튼 (상단)
- 일괄 작업: 선택한 작업 삭제, 클라이언트 변경

### 5.5 작업 상세 페이지 (Task Detail)
**경로**: `/tasks/:id`  
**목적**: 작업 편집, 메모 추가  
**주요 요소**:
- 제목 (인라인 편집)
- 설명 (마크다운 에디터)
- 클라이언트/프로젝트 드롭다운
- 예상 시간
- 상태 버튼 (미배정 → 배정 → 진행중 → 완료)
- 할당 날짜·시간 (있으면 표시, 수정 가능)
- 태그 (여러 개 선택)
- 우선순위
- 생성일, 수정일
- 삭제 버튼
- 메모 섹션 ("이 작업 관련 빠른 메모")

### 5.6 통계 페이지 (Statistics / Analytics)
**경로**: `/stats`  
**목적**: 생산성 분석, 청구 시간 확인  
**주요 요소**:
- 기간 선택 (주/월/분기)
- 섹션 1: 주간 작업량
  - 막대 그래프 (일별 배정 시간)
  - 라인 그래프 (누적 완료율)
- 섹션 2: 프로젝트별 시간
  - 원형 차트 (시간 비율)
  - 테이블 (프로젝트명, 예상 시간, 완료 시간, 완료율)
- 섹션 3: 클라이언트별 청구 시간 (프리랜서용)
  - 테이블 (클라이언트, 프로젝트, 시간, 단가, 청구액)
  - CSV 내보내기 버튼
- 섹션 4: 효율성 지표
  - 평균 완료 시간 (예상 vs 실제)
  - 이번 주 완료율 (%)
  - 평균 할당률 (배정된 시간 / 전체 시간)

### 5.7 설정 페이지 (Settings)
**경로**: `/settings`  
**목적**: 사용자 설정, 클라이언트 관리  
**주요 요소**:
- 섹션 1: 기본 설정
  - 사용자명, 프로필 이미지
  - 디폴트 근무시간 (예: 06:00~22:00)
  - 시간 단위 (30분, 1시간)
  - 주 시작일 (월요일/일요일)
- 섹션 2: 클라이언트/프로젝트 관리
  - 클라이언트 목록 (생성, 편집, 삭제)
  - 프로젝트 목록 (클라이언트별)
  - 색상 선택 (각 클라이언트)
  - 요금 설정 (선택사항: 시간당 요금)
- 섹션 3: 백업 & 내보내기
  - 데이터 백업 (JSON 다운로드)
  - 데이터 복원 (JSON 업로드)
- 섹션 4: 정보
  - 앱 버전
  - 오픈소스 라이센스

## 6. 데이터 모델 (초안)

### 6.1 테이블 설계

```sql
-- 클라이언트/프로젝트
CREATE TABLE clients (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  color TEXT NOT NULL, -- hex 색상 (#FF5733)
  hourly_rate REAL, -- 청구 요금 (선택)
  created_at INTEGER NOT NULL, -- unix timestamp
  updated_at INTEGER
);

CREATE TABLE projects (
  id INTEGER PRIMARY KEY,
  client_id INTEGER NOT NULL,
  name TEXT NOT NULL,
  description TEXT,
  color TEXT, -- 프로젝트별 색 (없으면 클라이언트 색 사용)
  created_at INTEGER,
  updated_at INTEGER,
  FOREIGN KEY (client_id) REFERENCES clients(id) ON DELETE CASCADE
);

-- 작업
CREATE TABLE tasks (
  id INTEGER PRIMARY KEY,
  project_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  description TEXT,
  estimated_hours REAL NOT NULL DEFAULT 1.0,
  status TEXT NOT NULL DEFAULT 'unassigned', 
    -- unassigned, assigned, in_progress, completed
  priority TEXT DEFAULT 'medium', -- low, medium, high
  created_at INTEGER NOT NULL,
  updated_at INTEGER,
  FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE
);

-- 할당 (작업을 캘린더에 배치)
CREATE TABLE task_assignments (
  id INTEGER PRIMARY KEY,
  task_id INTEGER NOT NULL UNIQUE,
  assigned_date TEXT NOT NULL, -- YYYY-MM-DD
  start_time INTEGER NOT NULL, -- 0~1440 (분 단위, 예: 14*60=840 = 14:00)
  end_time INTEGER NOT NULL,
  actual_duration REAL, -- 실제 소요 시간 (작업 완료 후 기록)
  completed_at INTEGER, -- unix timestamp
  created_at INTEGER,
  updated_at INTEGER,
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE
);

-- 이벤트 (회의, 약속 등)
CREATE TABLE calendar_events (
  id INTEGER PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  event_date TEXT NOT NULL, -- YYYY-MM-DD
  start_time INTEGER NOT NULL, -- 분 단위
  end_time INTEGER NOT NULL,
  color TEXT, -- 회의 색상 (기본: 회색)
  ical_uid TEXT, -- iCal 임포트 시 사용
  created_at INTEGER,
  updated_at INTEGER
);

-- 태그 (선택사항)
CREATE TABLE tags (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  created_at INTEGER
);

CREATE TABLE task_tags (
  task_id INTEGER NOT NULL,
  tag_id INTEGER NOT NULL,
  PRIMARY KEY (task_id, tag_id),
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);

-- 메모 (작업에 첨부되는 메모)
CREATE TABLE task_notes (
  id INTEGER PRIMARY KEY,
  task_id INTEGER NOT NULL,
  content TEXT NOT NULL, -- 마크다운 지원
  created_at INTEGER,
  updated_at INTEGER,
  FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE
);
```

### 6.2 데이터 흐름

```
사용자 → 작업 생성 → task (status='unassigned')
  ↓
  백로그 페이지 표시
  ↓
  드래그 앤 드롭 → task_assignments 생성
  ↓
  캘린더 그리드에 표시 (task + task_assignment)
  ↓
  작업 완료 → task 상태를 'completed', assignment.completed_at 기록
  ↓
  통계 페이지에서 분석
```

## 7. 기술 사양

### 7.1 스택
- **프론트엔드**: Next.js 16.2.9 (App Router)
- **UI 라이브러리**: shadcn/ui 2.9.0 (Radix + Tailwind v4)
- **스타일**: Tailwind CSS v4 (zero-config)
- **데이터베이스**: SQLite (better-sqlite3)
- **ORM**: Drizzle 0.31.5
- **드래그앤드롭**: dnd-kit (react-beautiful-dnd 호환성 검토 후 선택)
- **차트**: Recharts (가벼움, shadcn 호환)
- **마크다운**: markdown-it 또는 react-markdown

### 7.2 비기능 요구사항

#### 성능
- 캘린더 그리드 로드: < 500ms
- 검색 응답: < 200ms
- 드래그 앤 드롭 부드러움 (60fps)

#### 보안
- XSS 방지: 모든 사용자 입력 sanitize
- SQL Injection 방지: Drizzle 파라미터 바인딩
- 로컬 캘린더만 사용 → 외부 API 노출 없음

#### 신뢰성
- 데이터 손상 방지: WAL 모드 + 트랜잭션
- 오프라인 지원: 로컬 SQLite이므로 네트워크 불필요
- 백업 기능: JSON 내보내기 (설정 페이지)

#### 접근성
- WCAG 2.1 Level AA 준수
- 키보드 네비게이션 (TAB, ENTER, ESC)
- 색상 외 시각 표지 (아이콘, 텍스트 라벨)
- 화면 리더 지원 (ARIA 라벨)

#### 확장성
- Repository 패턴으로 데이터 접근 추상화
- 모듈식 컴포넌트 (각 화면 독립)
- iCal 임포트 후크 설계 (v1.1용)
